# Geographic lookup for Bay Area hospital-load maps

- **bayarea_cell_fips.csv** — per-grid-cell -> census-tract mapping for the Bay
  Area ExaEpi runs (71x70 grid, 4966 communities). Columns: `i,j,comm,geoid`.
  `geoid` = `f"{FIPS:05d}{Tract:06d}"` (11-digit 2000 census tract). Generated by
  reading the mesh fields `FIPS`, `Tract`, `comm` from `plt00000` of a 1-day Bay
  Area census run (the mapping is static geography, identical across all Bay Area
  runs / the same `BayArea.dat`).
- **bayarea_2000_tracts.{shp,...}** — 2000 TIGER census-tract polygons for the 9
  Bay Area counties (1405 tracts; San Francisco has 176, the 2000 count, not the
  197 of 2010). `BayArea.dat` is 2000-vintage, so every sim tract has a polygon
  here (the 1403 cell tracts are a subset of these 1405). Columns: `geoid` plus
  `STATEFP`/`COUNTYFP`/`TRACTCE`, so the file also drives
  `utilities/build_hospital_data.py` for the tract-level hospital data.

Used by `2026_MedicalWorkers/draft/figures/plot_h1_maps.py`.
